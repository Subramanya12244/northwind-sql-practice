# Q07. Employees Above Average Revenue

**Category:** GROUP BY & HAVING
**Difficulty:** Medium

---

## Problem Statement

Sales management wants to identify which employees are performing above the team's average revenue contribution, to recognize strong performers.

## Objective

Return all employees whose total revenue exceeds the average total revenue computed across all employees.

## Tables Used

- `employees`
- `orders`
- `order_details`

## Expected Output

| Column Name | Description |
|-------------|-------------|
| employee_id | Unique identifier of the employee |
| first_name | Employee's first name |
| last_name | Employee's last name |
| total_revenue | Sum of discounted revenue generated by this employee |

**Sample output:**

| employee_id | first_name | last_name | total_revenue |
|-------------|------------|-----------|---------------|
| 4 | Margaret | Peacock | 232890.85 |
| 3 | Janet | Leverling | 202812.84 |
| 1 | Nancy | Davolio | 192107.60 |
| 2 | Andrew | Fuller | 166537.76 |

*(Sample values are illustrative, based on the standard Northwind dataset, and intended to show shape/format — not guaranteed to match your exact data instance.)*

## Concepts Used

- INNER JOIN
- GROUP BY
- CTE
- Aggregate Functions (SUM, AVG)
- Subquery

## Why This Approach

**Why the same CTE-then-compare pattern as Q6:** the underlying problem structure is identical — 'above the average across the group' always requires computing per-entity totals first, then comparing each to an independently computed average over the full set.

## Common Mistakes

- Same as Q6 — attempting `HAVING SUM(x) > AVG(x)` directly instead of via an independently computed average.
- Grouping by name only instead of `employee_id`, risking incorrect merges if two employees share a name.

## Difficulty

**Medium**

## Interview Follow-up Questions

1. How is this query structurally identical to Q6, just applied to a different entity?
2. How would you generalize this into a reusable view or function if this 'above average revenue' check needed to be run for customers, employees, AND shippers?
3. What would change if 'average' should mean median instead of mean?

## Learning Outcomes

- Recognize and reuse the 'CTE + scalar subquery average comparison' pattern across different entities (customers, employees) — a key skill for writing maintainable SQL.
